# Claude Session — 2026-02-28

## Project: rodmachen.com

### Topic: Representing external publications on the site

User has stories published at Austin Chronicle (~140 articles) and Cinapse (~218 articles). Explored options for representing these on the site without copying content. Discussed five approaches: clips page with links, new content collection, publication landing pages, unified writing index, and curated highlights.

**Resolution:** Chose a new `bylines` content collection with manual markdown files. Each file contains frontmatter (title, subtitle, publication, url, date, tags) and the first paragraph of the published article as body content, ending with a "Read more at {publication}" link.

### Topic: Integrating bylines into the convert-to-blog plan

Incorporated the bylines feature into the existing `docs/plans/convert-to-blog.md` as Phase 6. Key decisions:

- **Collection name:** `bylines` (user preferred "byline" over "clips")
- **Navigation:** Renamed Blog to Writing, combining existing posts and external bylines into one unified hub
- **Categories updated:** Added `essay` as a new category/template alongside article, newsletter, review, and byline
- **Subtitles:** Added `subTitle` field to bylines schema (matching existing posts schema). Fetch scripts extract subtitles from both sources. Substack migration also captures subtitles.

### Topic: Scaling from ~25 to ~350+ byline files

Initial estimate of 20-25 bylines was corrected — Austin Chronicle has ~140 articles (12 pages) and Cinapse has 218 posts. Updated plan to use automated fetch scripts instead of manual creation:

- **`scripts/fetch-cinapse-bylines.mjs`** — Uses WordPress REST API (author ID 32, two-pass download matching cinapse-intelligence project approach). Extracts first paragraph from `content.rendered` HTML.
- **`scripts/fetch-chronicle-bylines.mjs`** — Two-step scrape: paginate author listing for URLs, then fetch each article page for subtitle and first paragraph. Uses cheerio for HTML parsing.
- **Idempotency** built in from the start — scripts check for existing files before writing, safe to re-run for incremental updates.

Referenced `/Users/rodmachen/code/cinapse-intelligence/cinapse-recs-plan.md` for WordPress API patterns.

### Topic: Removing duplicate published posts

Posts in `src/content/posts/` with `published: true` in frontmatter are copies of externally published pieces. These will be deleted in Phase 6 step 6d since they'll be replaced by byline entries linking to the originals. Only unpublished/original posts remain as articles.

### Topic: AI-powered tag generation

Added Phase 7 to the plan: a script (`scripts/generate-tags.mjs`) that uses Claude Haiku to assign tags from a controlled vocabulary to all content (~400+ files across posts and bylines). Includes dry-run mode. Cost under $1. Controlled vocabulary ensures consistency (e.g., "film" not "movies"/"cinema").

### Topic: Read More links in byline body content

Updated the byline markdown format to include a `[Read more at {publication} →](url)` link baked into the markdown body by the fetch scripts. Applies to all bylines from both Cinapse and Chronicle. Renders naturally on listing pages without special template logic.
